Resource paths in monkey.

App resources may be stored in many different locations, for example, in an app's data dir, on the local filesystem, or on a remote server.

Methods for opening and loading resources typically require the use of a 'path' parameter. You may use any of the following path prefixes when using such methods:

  • Use the "./" prefix to locate resources stored relative to the current directory, or relative to the document URL.

  • Use the "/" prefix to locate resources stored relative to the root directory of the current directory, or relative to the root directory of the document URL.

  • Use the "http://" or "https://" prefixes to locate resouces stored on a server.

  • Use the "monkey://data/" prefix to locate resources that were placed in the app's data/ directory when it was built.

  • Use the "monkey://local/" prefix to locate resources stored in the app's local storage directory. This directory will initially be empty when the app is first installed. Currently only available on the ios and android targers.

    • Not all prefixes work on all targets and the exact behavior of each prefix is target dependant. For example, android and ios can load images from http:// but not audio, while glfw cannot load from http:// at all. Full support for all prefixes is an ongoing mission!

    Also, note that Mojo module load commands will automatically insert a "monkey://data/" prefix before any path that does not already containing a prefix.
